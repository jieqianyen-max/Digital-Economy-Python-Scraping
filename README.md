# 📊 数字经济数据采集与分析实战项目

> **项目目标**：对标金融科技与数字经济标准，实现从静态网页到动态 API 接口的全自动化数据采集与资产化。

## 🚀 已实现的技术突破 (Technical Milestones)

### 1. 破解反爬拦截
* **Debugger 破解**：通过浏览器开发者工具（DevTools）禁用硬编码断点，成功绕过雪球网等主流金融网站的 `debugger` 防调试陷阱。
* **身份伪装**：封装 `User-Agent` 与 `Headers` 请求头，模拟真实用户浏览器行为。

### 2. 深层数据接口嗅探 (API Sniffing)
* 弃用低效的 HTML 网页解析（BeautifulSoup）。
* 深入 **Network (网络)** 面板，直接抓取后端 `JSON` 异步数据接口，实现极速、纯净的数据获取。

### 3. 数据资产自动化转化
* 使用 **Pandas** 库对抓取的非结构化数据进行清洗。
* 实现数据自动导出为 `.xlsx` (Excel) 格式，形成可直接用于商业分析的“数字资产”。

### 4. 会话身份伪装：
* 掌握了 Cookie 注入 技术，实现了带权接口的模拟登录访问，突破了游客访问频率限制。

### day4进阶：构建工业级多目标监控系统:
* 自动化升级：从单兵作战演进为基于 Dict 和 For-Loop 的多目标并行采集。

* 逆向排雷：独立解决了 URL 路径漂移导致的 403/404 异常，深刻理解了 HTTP 状态码的业务含义。

* 防御性编程：引入了本地映射表与 .get() 容错机制，确保了在接口数据残缺时的系统稳定性。

* 成果：成功产出《Day4_全平台监控资产_完美版.xlsx》，实现分钟级实时行情抓取。

### day5新增模块：## 🤖 智能 Agent 预警系统。

技能点：实时轮询 (Polling)、逻辑决策逻辑 (Decision Logic)、频率限制 (Rate Limiting)。

### Day 8 开启动态爬取学习，成功搭建 Selenium 环境并实现模拟搜索。

### Day 9：金融数据自动化落地与 ETL 实战
* 1. 今日目标

实现从雪球行情中心自动抓取新股申购数据。

攻克 JavaScript 异步加载导致的字段缺失问题。

将动态网页数据结构化为 CSV 文件，建立个人金融数据库的“第一块砖”。

* 2. 解决的实战问题 (Troubleshooting)

问题：初始抓取时，“最新价”和“涨跌幅”字段为空。

原因：网页采用了 Ajax 异步渲染，数据加载慢于脚本执行。

对策：引入 time.sleep 缓冲并切换至基于 td 标签的精确索引定位。

* 3. 产出物 (Artifacts)

生成了 Market_Snapshot_20260408.csv，包含股票代码、名称、发行价等核心特征。

### Day 10 
* 成功实现雪球 L1 级逐笔成交数据的 API 逆向采集。攻克了 latin-1 编码冲突和 Unix 时间戳转换。目前已建立自动化的股票特征采集引擎。

### Day 11 
* 利用 Pandas 实现了宁德时代逐笔成交数据的深度清洗：完成了 Unix 时间戳转换、买卖盘特征映射及成交金额的向量化计算。成功将原始 API 响应转化为标准化的金融分析面板。

### Day 12 
* 实现了从单股分析到多股面板（Portfolio Panel）的跨越。通过 pd.concat 整合了 5 只龙头股的实时成交数据，并初步实现了基于 symbol 的分组均价计算。"

### Day 13: 资金流向因子构建与 Groupby 聚合实战

* 总结：今天利用 Pandas 实现了多标的资金流向计算。通过特征工程将原始成交 side 转换为净流入金额，并利用 groupby 实现了全市场截面分析。这是我第一个真正意义上的量化分析指标。


### Day 14 全市场资金流向监控系统
* 自动化流水线：构建了支持 50 只热门股并发采集的 Python 引擎，具备重试机制与 IP 防封保护。

* 量化因子建模：利用 Pandas 计算出核心因子 Flow Ratio（净流入占比），实现了从原始成交数据到投资信号的转化。
